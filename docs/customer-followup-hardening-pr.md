## المشكلة

مساحة متابعة العملاء تعتمد على قواعد متكررة للهوية والهاتف والحالات، وسكربت قديم يعدل المصدر أثناء البناء، مع نقص في اختبارات السيناريوهات الحرجة.

## السبب الجذري

- اختلاف ترتيب هوية العميل بين مصادر البيانات.
- أكثر من دالة لتوحيد الهاتف.
- قواعد الحالات موزعة داخل الواجهة.
- مشغل الاختبارات لا يكتشف كل الاختبارات.
- إصلاحات Text Replace تعمل ضمن dev/build.

## التنفيذ الجاري

- طبقة موحدة لهوية العميل والهاتف والحالات والنشاط ومقدم الطلب.
- تشخيص جودة بيانات موحد.
- اختبارات Unit وIntegration أولية.
- اكتشاف تلقائي لكل ملفات الاختبار.
- View تشخيصية آمنة للتكرارات المفتوحة بدون حذف بيانات.

## قبل الدمج

- ربط الطبقة المشتركة بكل المسارات.
- إزالة سكربت تعديل المصدر من Pipeline.
- Pagination وتصدير كامل.
- RPC آمنة لمنع إنشاء Case مفتوح مكرر.
- نجاح doctor/typecheck/lint/test/build وgit diff --exit-code.
