### 2. تحديث تصنيف العملاء

**المشكلة:** تصنيف العملاء يعتمد على بيانات وهمية والقيم غير صحيحة.

**القيم القديمة (في normalizeCustomerSegment):**
- totalSpent >= 20000 أو avgMonthly >= 5000 → مهم جدًا
- totalSpent >= 10000 أو avgMonthly >= 2500 → مهم
- totalSpent >= 3000 أو avgMonthly >= 800 → متوسط

**القيم الجديدة (المطلوبة):**
- avgMonthly >= 8000 → مهم جدًا
- avgMonthly >= 4000 → مهم
- avgMonthly >= 1500 → متوسط
- avgMonthly < 1500 → عادي

**الإصلاح:**
- تحديث دالة `normalizeCustomerSegment` في `src/lib/customerAnalyticsService.ts` لاستخدام القيم الجديدة
- إزالة الاعتماد على `totalSpent` والاعتماد فقط على `avgMonthly`

**الملفات المعدلة:**
- `src/lib/customerAnalyticsService.ts`

### 3. إنشاء دالة تحديث تصنيف العملاء تلقائياً

**المشكلة:** التصنيف لا يتم تحديثه تلقائياً من فواتير المبيعات.

**الحل:** إنشاء دالة `syncCustomerSegmentationFromInvoices` تقوم بـ:
1. جلب جميع الفواتير من جدول `sales_invoices`
2. جلب جميع العملاء من جدول `customers`
3. حساب متوسط المشتريات الشهرية لكل عميل
4. تحديد التصنيف المناسب بناءً على القيم الجديدة
5. تحديث بيانات العملاء في قاعدة البيانات

**الإصلاح:**
- إضافة دالة `syncCustomerSegmentationFromInvoices` في `src/lib/customerAnalyticsService.ts`
- الدالة تحسب `avg_monthly` من الفواتير وتحدد التصنيف المناسب
- الدالة تحديث `segment`, `avg_monthly`, `total_purchases`, `total_invoices`, `last_purchase`

**الملفات المعدلة:**
- `src/lib/customerAnalyticsService.ts`

## 📊 كيفية استخدام دالة تحديث التصنيف

يمكن استدعاء الدالة من أي مكان في التطبيق:

```typescript
import { syncCustomerSegmentationFromInvoices } from "@/lib/customerAnalyticsService";

// تحديث تصنيف العملاء

if (result.success) {
  console.log(`تم تحديث ${result.updated} عميل`);
} else {
  console.error(`خطأ: ${result.error}`);
}
```
