# تقرير التعديلات النهائية - Dawaa Pharmacy 2027

تاريخ التجهيز: 2026-07-07

## نتيجة الفحص

- تم تشغيل `npm ci --legacy-peer-deps --no-audit --no-fund` بنجاح داخل بيئة الاختبار.
- تم تشغيل `npm run typecheck` بنجاح بعد إصلاح خطأ TypeScript في `userDataScope.ts`.
- تم تشغيل `npm run build` بنجاح.
- لا توجد أخطاء TypeScript أو Build في النسخة المرفقة.

## أهم التعديلات المنفذة

### 1. تنظيف القائمة الجانبية
- تثبيت صفحة ملاحظات الشيفت أعلى القائمة مع badge لعدد الملاحظات المفتوحة.
- إزالة تكرار ملاحظات الشيفت من الأقسام الداخلية.
- إخفاء مركز القيادة اليومي وملخص اليوم من القائمة بدون حذف الكود.
- إخفاء CRM وCustomer 360 وطلبات العملاء من القائمة بدون حذف الكود.
- إخفاء مجموعة الدليفري من تطبيق الإدارة عبر feature flag: `ENABLE_INTERNAL_DELIVERY_MODULE = false`.
- تعديل متابعة العملاء لتفتح صفحة خدمة العملاء بدل صفحة طلبات العملاء.
- نقل مرور مدير الفروع أسفل لوحة القيادة مباشرة.
- تبسيط قائمة الدكتور لتصبح موجهة للاستخدام اليومي.

### 2. صلاحيات ونطاق بيانات المستخدم
- دعم override د/ ضحى بحيث dashboardBranch = فرع الشامي.
- دعم عرض تقييمات/واتساب فرعي الشامي وشكري لد/ ضحى ود/ دنيا.
- منع استخدام المخزن كفرع تحليلات لخدمة العملاء.
- إصلاح typing في `ScopeUser` حتى تقبل الدوال بيانات المستخدم الجزئية بأمان.

### 3. حل تكرار الدكاترة وهوية الموظف
- إضافة/تطوير `src/lib/staff/staffIdentityResolver.ts`.
- إضافة تطبيع أسماء الدكاترة وإزالة د/ وdr وتوحيد الأسماء العربية.
- دعم aliases مثل إسلام / اسلام / Eslam / Islam وربطها بالحساب الأساسي النشط.
- توفير helpers مثل `resolvePrimaryStaffForDoctor` و`buildStaffIdentityMap` لاستخدامها في التحليلات والمسابقات.

### 4. WhatsApp Analytics
- ربط صفحة تحليلات واتساب بصلاحيات المستخدم والفروع.
- دعم عرض كل الفروع للمدير العام/مدير الفروع/مدير خدمة العملاء فقط.
- دعم فرعي الشامي وشكري لد/ ضحى ود/ دنيا.
- منع ظهور كل الفروع للمستخدم غير المصرح له.
- تجميع النتائج حسب الموظف الأساسي بدل الاسم فقط عند توفر بيانات الربط.

### 5. مسابقة الدكاترة
- تعزيز فصل المسابقة حسب الفرع.
- استخدام helpers الهوية لتقليل تكرار أسماء الدكاترة.
- حماية أفضل ضد عرض بيانات فرع آخر للدكتور العادي.

### 6. صفحة الدكتور
- تحسين صفحة الدكتور لتعرض ملخصًا أقوى عن النقاط والمبيعات والعملاء والرواكد والحوافز.
- تحسين حماية النطاق بحيث لا يرى الدكتور إلا نفسه أو فرعه حسب صلاحياته.

### 7. مركز التقارير
- تحويل التصدير إلى XLSX باستخدام مكتبة `xlsx`.
- إضافة أسماء ملفات عربية واضحة تحتوي التقرير والفرع والفترة.
- دعم تقارير أكثر مثل المبيعات، الرواتب، العملاء المتوقفين، أفضل العملاء، النواقص، تقييمات المحادثات، واتساب، أداء الدكاترة، الرواكد، الحوافز، والتقرير الشهري.

### 8. تكويد العملاء
- إضافة تصدير Excel لطلبات تكويد العملاء.
- التصدير يحترم الفلاتر الحالية: البحث، الحالة، الفرع.
- الملف يحتوي بيانات العميل، الهاتف، العنوان، الفرع، الحالة، المصدر، الملاحظات وتواريخ المتابعة.

### 9. استيراد الفواتير
- إضافة `src/lib/invoices/invoiceAccess.ts` لتنظيم صلاحيات صفحة استيراد الفواتير.
- السماح بالوصول للصفحة للأسماء/الحسابات المحددة: د أميرة، د علياء، د معاذ، د علا.
- صلاحية حذف batches مقصورة على د معاذ ود علا فقط، مع بقاء المدير العام بصلاحيات الإدارة.

## ملاحظات مهمة قبل النشر

1. النسخة لا تحتوي `node_modules` ولا `dist` ولا ملفات البيئة المحلية الحساسة مثل `env.local`.
2. ارفع الملفات على GitHub ثم اترك Vercel يبني من جديد.
3. اختبر بعد النشر:
   - القائمة الجانبية.
   - حساب دكتور عادي.
   - حساب د/ ضحى.
   - حساب د/ دنيا.
   - حساب د/ معاذ أو د/ علا.
   - تحليلات واتساب.
   - مسابقة الدكاترة.
   - تقارير Excel.
   - تكويد العملاء export.
   - استيراد الفواتير وصلاحيات الحذف.
