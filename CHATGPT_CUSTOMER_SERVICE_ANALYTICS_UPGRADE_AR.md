# تقرير تعديل تحليلات المبيعات ومركز خدمة العملاء

## النتيجة
- تم تشغيل `npm run typecheck` بنجاح.
- تم تشغيل `npm run build` بنجاح.
- البناء الإنتاجي نجح في بيئة الاختبار داخل الحاوية.

## الملفات المعدلة
1. `src/pages/Analytics.tsx`
2. `src/lib/dashboardSummaryService.ts`
3. `src/lib/api/customerServiceCommandCenter.ts`
4. `src/pages/CustomerService.tsx`

## ما تم تنفيذه في تحليلات المبيعات
- سبب المشكلة كان أن View/RPC الخاصة بمؤشرات العملاء `dawaa_dashboard_customer_cards_v13` قد تعمل ببطء وتسبب `statement timeout`.
- تم منع فشل مؤشرات العملاء من إسقاط صفحة التحليلات بالكامل.
- تم فصل تحميل مؤشرات V13 إلى أقسام مستقلة:
  - مؤشرات العملاء.
  - متوسط الفرع/الشيفت.
  - تارجت الفروع.
- تم إضافة timeout آمن على مستوى الواجهة لكل مصدر V13.
- عند فشل كارت العملاء تظهر رسالة واضحة بدل عرض أصفار مضللة.
- تم إضافة حماية في `dashboardSummaryService` حتى لا ينتظر مصدر العملاء V13 إلى أجل غير محدود.

## ما تم تنفيذه في مركز خدمة العملاء
- إضافة قوائم تحليلية جديدة داخل مركز خدمة العملاء:
  - متابعة قوية.
  - أهم العملاء.
  - عملاء قللوا التعامل.
  - عملاء توقفوا أكثر من شهرين.
  - أثر المتابعات.
  - أداء مسؤولي خدمة العملاء.
- إضافة منطق يمنع تكرار العميل داخل القوائم التحليلية باستخدام كود العميل/الهاتف/معرف العميل.
- إضافة زر `متابعة قوية` داخل كارت العميل لتحويل العميل بسرعة إلى متابعة قوية.
- عند تحويل العميل إلى متابعة قوية يتم تجهيز نموذج المتابعة ببيانات العميل وسبب المتابعة وتاريخ المتابعة.
- تحسين عرض كارت العميل بإظهار `المتابعة القادمة` حتى تعرف الدكتورة موعد الرجوع للعميل.
- إضافة تحميل مستقل للقوائم التحليلية مع رسالة واضحة لو تعذر تحميلها.
- إضافة منطق توزيع خدمة العملاء:
  - `cs.doha` / د ضحى ترى فرع الشامي في مركز خدمة العملاء.
  - `cs.donia` / د دنيا ترى فرع شكري في مركز خدمة العملاء.
  - المديرون يحتفظون بإمكانية فلترة كل الفروع.
- تعديل إنشاء قائمة اليوم لتكون قائمة ذكية حتى 20 عميل لكل فرع، مع منع تكرار العملاء الذين لديهم متابعة مفتوحة أو موجودون في نفس اليوم.
- أولوية القائمة اليومية أصبحت:
  1. العملاء المتوقفون أكثر من شهرين.
  2. العملاء الذين قللوا التعامل.
  3. أهم العملاء الحاليين.

## ملاحظات تشغيل مهمة
- لم يتم حذف أي بيانات.
- لم يتم حذف أي صفحات أو Routes.
- لم يتم عمل Migration جديد في هذه المرحلة.
- إذا استمر بطء `dawaa_dashboard_customer_cards_v13` في Supabase، يفضل لاحقًا إضافة indexes أو materialized summary مخصص، لكن الواجهة الآن لن تسقط بسبب timeout.
