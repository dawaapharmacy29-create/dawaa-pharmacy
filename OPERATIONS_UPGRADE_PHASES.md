# تطوير نظام تشغيل صيدليات دواء

الفرع التنفيذي: `feature/full-operations-upgrade`

## معايير إلزامية

- عدم كسر بيانات الفواتير أو العملاء أو الموظفين الحالية.
- كل قسم تحميل مستقل: `loading / error / loadedAt`.
- لا تعرض قيمة صفر عند فشل التحميل؛ اعرض حالة الخطأ بوضوح.
- تسجيل أي تعديل إداري مهم في `activity_log`.
- الدورة الشهرية من يوم 26 حتى يوم 25.
- صلاحيات العرض حسب الفرع والدور مع منع تسريب بيانات الفروع.

## الدفعة 1 — الحضور والفريق

- [ ] تقسيم الموجودين في الشيفت حسب الفرع.
- [ ] داخل كل فرع: الدكاترة والصيادلة، المساعدون، ثم الدليفري.
- [ ] إخفاء عبارة «لم يبصم» من العرض العام، وإظهار الحالات المهمة فقط.
- [ ] أخذ ثلاث قراءات GPS واختيار الأدق.
- [ ] عدم رفض الحضور على الدقة وحدها إذا كان الموظف داخل نطاق الفرع.
- [ ] إضافة حالة مراجعة يدوية للحالات الحدودية.

## الدفعة 2 — الحسابات والصلاحيات

- [ ] مصدر آمن موحد لعرض الحسابات.
- [ ] المدير العام ومدير الفروع يشاهدان جميع الحسابات.
- [ ] فلاتر الفرع والدور وحالة الدخول والربط بالموظف.
- [ ] تعديل الدور والفرع والصلاحيات وتفعيل/تعطيل الدخول.
- [ ] كشف الحسابات المكررة والحسابات بدون موظف.

## الدفعة 3 — الإجازات والمهام

- [ ] صفحة الإذونات Responsive بدون الحاجة إلى Zoom Out.
- [ ] اعتماد ورفض وإرجاع الإجازة للمراجعة.
- [ ] تسجيل المعتمد والوقت ونوع الخصم والرصيد.
- [ ] ظهور الإجازة المعتمدة في التقرير الشهري للموظف.
- [ ] إنشاء مهام اليوم بصورة idempotent بدون تكرار.
- [ ] To-Do List لمدير الفرع مع المسؤول والأولوية والموعد والإثبات والاعتماد.

## الدفعة 4 — التحليلات والتقارير

- [ ] معالجة `statement timeout` باستخدام ملخصات مجمعة وPagination.
- [ ] إصلاح مصدر `customer_metrics` أو توفير fallback صحيح.
- [ ] تقارير Excel دقيقة متعددة الأوراق.
- [ ] تصدير تحليل واتساب شامل بالمعدلات والتوصيات التدريبية.

## الدفعة 5 — سجل الأنشطة والنظافة وصحة البيانات

- [ ] إصلاح RLS لسجل الأنشطة حسب الدور والفرع.
- [ ] تسجيل تعديلات الحسابات والإجازات والمهام والفواتير والعملاء.
- [ ] إظهار صفحة النظافة وربط route/sidebar/permissions.
- [ ] معالجة العملاء بدون فرع أو هاتف والفواتير بدون عميل.
- [ ] Staging + Preview + Audit قبل تحديث بيانات العملاء.

## بوابة الدمج

قبل الدمج إلى `main` يجب نجاح:

```bash
npm run typecheck
npm run test --if-present
npm run build
```
